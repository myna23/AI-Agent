Zambia GeoHub AI — System Architecture

Three-layer backend + Streamlit frontend  ·  World Bank Technical Review

1
Data Layer
Pulls live geospatial features from Zambia GeoHub via ArcGIS FeatureServer REST API
📦 Catalog Discovery — Dataset Index
At startup, two HTTP GET requests hit the ArcGIS Online search API to build the full dataset index. GET https://www.arcgis.com/sharing/rest/search ?q=tags:zmb type:"Feature Service" &num=100 &f=json GET https://www.arcgis.com/sharing/rest/search ?q=orgid:iQ1dY19aHwbSDYIF type:"Feature Service" &num=100 &f=json Results are filtered for Zambia keywords, broken hosts excluded, and merged with a hardcoded seed catalogue of ~40 curated entries. Cached for the life of the server process.
🔢 Exact Count Query — returnCountOnly
Before fetching any geometry, a lightweight count-only request returns an exact integer. No features, no geometry — minimal payload. GET {featureUrl}/query ?where=District='Lusaka' &returnCountOnly=true &f=json → Response: { "count": 312 } This integer is injected into the AI prompt as a verified fact. The AI is instructed to state it directly without hedging.

WHERE clause construction:
— District query → District='{name}' OR DISTRICT='{name}'
— Province query → Province='{name}' OR PROVINCE='{name}'
— POI type query → Type='Religion' or Type='Commercial'
📡 Feature Fetch — 4-Tier Cascade
The same ArcGIS endpoint is tried four ways in sequence. Each tier is only attempted if the previous returned zero features.
Tier 1 — ArcGIS SDK (primary) from arcgis.features import FeatureLayer fl = FeatureLayer(url) result = fl.query( where=where, out_fields="*", result_record_count=30 ) Official Esri Python SDK. Sends trusted headers recognised by ArcGIS servers that reject plain requests from cloud datacenter IPs.
Tier 2 — Raw HTTP, GeoJSON format GET {featureUrl}/query ?where=1=1 &outFields=* &resultRecordCount=30 &f=geojson &Referer: zmb-geowb.hub.arcgis.com Hard cap of 30 features — keeps payloads under ~60 KB on Streamlit Cloud. Referer header set on every session request.
Tier 3 — Plain JSON, no geometry GET {featureUrl}/query ?where=1=1 &outFields=* &returnGeometry=false &f=json Some ArcGIS servers refuse GeoJSON from cloud IPs but serve plain JSON. The attributes objects are converted to synthetic GeoJSON features with geometry: null.
Tier 4 — Static file fallback _STATIC_MAP = { "GRID3_ZMB_HealthFac": "health_facilities.json", "GRID3_ZMB_School": "schools.json", "GRID3_Zambia_Operational_Settlement_Points": "settlements.json", ... # 16 files total } Pre-downloaded GeoJSON files committed to the repo. District/province filter applied in Python. Returns _source: "static_sample" → UI shows yellow "Static data" badge.
🗺️ Spatial Count (Draw Area) — Overpass API for OSM features
For the Draw Area feature, 5 datasets (Mines, Dams, Churches, Mosques, Markets) are queried from OpenStreetMap via the Overpass API because the ArcGIS versions of these layers have geometry types (MultiLineString, Polygon) that don't resolve cleanly for bounding-box point counting. POST https://overpass.kumi.systems/api/interpreter data=[out:json][timeout:20]; (node["amenity"="place_of_worship"] ["religion"="christian"](S,W,N,E); way ["amenity"="place_of_worship"] ["religion"="christian"](S,W,N,E); );out count; → Response: elements[0].tags.total = 13 Three mirrors tried in sequence — first HTTP 200 wins: overpass.kumi.systems → overpass-api.de → maps.mail.ru. If count > 0, a second request with out body 20; fetches names and coordinates.
↓ GeoJSON features + exact count passed to AI Layer
2
AI Layer
Assembles a structured, data-grounded prompt and streams the AI response — supports Anthropic, OpenAI, and Google Gemini
🧭 Intent Classification & Dataset Ranking
Every message is classified before any API call is made — purely in Python, no AI used. # Intent detection if "generate a report" in query: → report_mode() elif "summarise" in query: → summariser_mode() elif "what data is available" in query: → catalogue_mode() else: → chatbot_mode() # Dataset ranking — scores every catalog entry for dataset in catalog: for token in query_tokens - STOP_WORDS: if token in dataset["name"]: score += 2 if token in dataset["tags"]: score += 2 if token in dataset["description"]: score += 1 score += SUBJECT_BOOST.get(dataset_url_fragment, 0) # SUBJECT_BOOST maps ~30 keywords to dataset URLs: # "flood" → Zambia_Flood_Prone_Districts # "mine" → zmb_mines_osm_20251009py # "poverty" → zmb_ADM3_const_Poverty → return sorted(catalog, key=score)[:5]
🔀 Provider Router — ModelClient
A single ModelClient class abstracts all three providers behind an identical interface. The user selects provider and pastes their API key in the Settings panel — stored in st.session_state only, never written to disk. class ModelClient: def stream_with_history(self, system, messages): if self.provider == "Anthropic (Claude)": with client.messages.stream( model=self.model, max_tokens=2048, system=system, messages=messages ) as s: yield from s.text_stream # token-by-token elif self.provider == "OpenAI (GPT)": stream = client.chat.completions.create( model=self.model, stream=True, messages=[{"role":"system","content":system}] + messages ) for chunk in stream: yield chunk.choices[0].delta.content elif self.provider == "Google (Gemini)": chat = model.start_chat(history=history) for chunk in chat.send_message( last_msg, stream=True): yield chunk.text On HTTP 429 (rate limit) or 529 (overloaded): exponential backoff, up to 3 retries at 3 s / 6 s / 9 s intervals.
📝 Prompt Constructor — What the AI Actually Receives
The user prompt is a structured string built entirely in Python before the AI API call. The AI never sees the raw GeoJSON — it receives pre-processed records and aggregations.
System prompt (rules, sent every call): "You are an AI assistant for the Zambia GeoHub. RULES: - Answer ONLY from the data provided to you. - Never invent statistics, field names, or facts. - If ⚡ EXACT COUNT is provided, state it directly. - If _note field says unavailable → server status only. - Always cite the dataset name used. - Never guess or invent a URL. - Do NOT add follow-up suggestions."
User prompt (assembled per query): Question: {user text} Matched datasets (up to 5): Dataset 1: GRID3 ZMB Health Facilities Description: ... Fields: Facility_N, Type, District, Province Source: zmb-geowb.hub.arcgis.com/... ⚡ EXACT COUNT: 312 records in Lusaka District Sample records (up to 15 features): [{"Facility_N":"Chilanga HC", "Type":"Health Facility", "District":"Chilanga", "Province":"Lusaka"}, ...] Aggregated counts: By District: Lusaka:42, Kafue:18 ... By Type: Health Facility:280, Hospital:32 Cross-context: flood data, risk layers, road count, settlement count All available datasets: [full list]
Aggregations (count by District, Province, Type) are computed in Python before the prompt — the AI receives the answer, not raw rows it has to count itself.
↓ Streamed AI response + GeoJSON passed to Map Layer
3
Map Layer
Renders interactive Leaflet maps, computes spatial analytics, and applies geometric algorithms for the Draw Area feature
🗺️ Folium + Leaflet.js Rendering
Each query result renders a Folium map (Python wrapper for Leaflet.js), embedded in Streamlit via streamlit-folium. Map is only built when the user clicks the Map button — not on every rerun. import folium m = folium.Map( location=[-13.5, 28.5], # Zambia centre zoom_start=6, tiles="cartodb-darkmatter" # dark basemap ) # Point features → CircleMarker for feat in geojson["features"]: props = feat["properties"] lat, lon = feat["geometry"]["coordinates"][1,0] color = _point_color(props, dataset_name) folium.CircleMarker( location=[lat, lon], radius=6, color=color, fill=True, fill_opacity=0.8, popup=folium.Popup(html_table(props)) ).add_to(m) # Polygon features → GeoJson layer folium.GeoJson( polygon_geojson, style_function=lambda f: { "color": theme["border"], "fillColor": theme["fill"], "weight": 1, "fillOpacity": 0.35 } ).add_to(m) Tile layer: CARTO Dark Matter — dark background makes coloured data points stand out.
Context layers loaded at startup: district boundary polygons + major roads always rendered underneath data points.
🎨 Colour Assignment System
Each feature's colour is determined by a 3-level lookup chain: # 1. POI Type field (most specific) _POI_TYPE_COLORS = { "Commercial": "#2d6a4f", # green "Religion": "#7b2d8b", # purple "Farm": "#90be6d", # light green "Mining": "#495057", # grey ... } # 2. Facility type keyword in Facility_T field # "hospital" / "clinic" / "health" → #e63946 red # "school" / "education" → #457b9d blue # 3. Dataset name keyword fallback _TYPE_COLORS = { "health": "#e63946", "school": "#457b9d", "market": "#2d6a4f", "mine": "#495057", "flood": "#4361ee", "forest": "#2d6a4f", "road": "#d4790a", "railway": "#9d0208", "power": "#f8961e", "poverty": "#7b2d8b", "default": "#e63946" } This ensures visual consistency — health is always red, schools always blue, markets always green — regardless of which dataset the feature came from.
📐 Mathematical & Geometric Models
Haversine Formula — Ground Distance
Used to calculate real spherical earth distance between two lat/lon points. Applied for: nearest feature to centroid, radius filtering, area computation. def haversine_km(lat1,lon1,lat2,lon2): R = 6371.0 # Earth radius km φ1 = radians(lat1) φ2 = radians(lat2) Δφ = radians(lat2 - lat1) Δλ = radians(lon2 - lon1) a = sin(Δφ/2)² + cos(φ1)·cos(φ2)·sin(Δλ/2)² return R · 2·atan2(√a, √(1-a))
Ray-Casting Algorithm — Point in Polygon
Used to determine which district(s) a drawn bounding box intersects. Tests 5 points (4 corners + centroid) against every district boundary ring using the Jordan Curve / even-odd rule. def _point_in_polygon(lat, lon, ring): # ring = [[lon,lat], ...] coordinate pairs inside = False j = len(ring) - 1 for i in range(len(ring)): xi,yi = ring[i][0], ring[i][1] xj,yj = ring[j][0], ring[j][1] if ((yi>lat) != (yj>lat)) and \ lon < (xj-xi)*(lat-yi)/(yj-yi)+xi: inside = not inside j = i return inside # even-odd rule
Draw Area Analytics — Derived Metrics
All 5 analytics computed from count data alone — no AI call required. # Area (rectangular approx) area_km2 = haversine_km( min_lat,min_lon,min_lat,max_lon) * haversine_km( min_lat,min_lon,max_lat,min_lon) # Population estimate pop = settlements * 6 # avg HH size # Healthcare access ratio ratio = (health / pop) * 10_000 # School coverage school_cov = schools / settlements # Area classification density = (settlements / area_km2) * 100 if density >= 50: class = "Urban" elif density >= 15: class = "Peri-urban" else: class = "Rural"
⚡ Parallel Spatial Queries — ThreadPoolExecutor
When the user clicks "Count features in area", all 9 queries (4 ArcGIS + 5 Overpass) fire simultaneously using Python's concurrent.futures.ThreadPoolExecutor. Total wait time = slowest single query, not the sum of all. with ThreadPoolExecutor(max_workers=9) as pool: arcgis_futures = [ pool.submit(_fetch_arcgis, Health_url, bbox), pool.submit(_fetch_arcgis, Schools_url, bbox), pool.submit(_fetch_arcgis, Roads_url, bbox), pool.submit(_fetch_arcgis, Settlements_url, bbox), ] osm_futures = [ pool.submit(_fetch_osm, "Mines", mine_query), pool.submit(_fetch_osm, "Dams", dam_query), pool.submit(_fetch_osm, "Churches", church_query), pool.submit(_fetch_osm, "Mosques", mosque_query), pool.submit(_fetch_osm, "Markets", market_query), ] for future in arcgis_futures + osm_futures: label, count, details = future.result() # Each returns: count, names[], nearest+distance, subtypes
↓ Folium map object passed to Streamlit frontend for rendering
4
Frontend Layer — Streamlit
UI framework only · Not part of the core backend · Serves as the interface between user and the three backend layers
⚙️ Session & State
Streamlit re-executes the full Python script on every user interaction. st.session_state persists: chat history, GeoJSON per message, draw bbox, language selection, API key (memory only — never written to disk or server).
📤 Output Components
Chat answer streamed via st.write_stream() · Map embedded via st_folium() · Table via st.dataframe() · Charts via Plotly · Downloads via st.download_button() (Word + PDF bytes)
🌐 Deployment
Hosted on Streamlit Community Cloud. Pulls directly from github.com/myna23/AI-Agent main branch. Stateless container — no persistent disk. Auto-redeploys on every git push.
